Audit Overview

Five Miva Merchant templates were reviewed for performance, SEO, accessibility, and code quality issues. Issues are categorised by severity and mapped to the exact template and line where the fix needs to be applied.

Root Cause — Why Global Issues Appear on Every Page

Every template calls <mvt:item name="hdft" param="global_header" /> and global_footer. This means scripts loaded in the hdft module (GTM, Klaviyo, Affirm, SearchSpring) fire on ALL pages. The hdft template itself was not accessible for review but its effects are visible in every template's console output. Fixing hdft will have the highest global impact.

Section 02

SFNT — Storefront (Homepage)

🏠

Template: SFNT — Storefront

Homepage · 133 lines · Cache: basket empty + no customer logged in · HTTPS enforced

Issue 1 — Affirm JS fires on homepage via global_header
CriticalPerformanceConsole Error
The Affirm "As Low As" module is set to Page Type: Homepage in SFNT settings. This causes Affirm's JS (145KB) to load on the homepage and throws a SecurityError: Failed to set the 'cookie' property — sandboxed iframe lacks allow-same-origin on every page load. Affirm is broken for most users due to Chrome's third-party cookie deprecation.
📍 SFNT → Affirm As Low As Messaging section → Page Type setting

Fix — No code change needed

Affirm As Low As Messaging
Page Type: Homepage ← CHANGE TO: None
Page Type: None ← saves 145KB, removes SecurityError
Issue 2 — Slick carousel: slideWidth:0 on indexes 7–17
CriticalCLS
Console shows Invalid slide dimensions: index 7, 8, 10–17 slideWidth: 0. This means sfnt_hero content section has empty/broken slides with no image assigned. Slick calculates zero width for these and they cause layout shift (contributing to CLS = 0.27 in real user data).
📍 Content Management → Theme Components → sfnt_hero → find slides with no image → delete them

Fix — In Theme Components, not the template

// Go to: Content Management → Theme Components → sfnt_hero
// Find each slide that has no image assigned
// Slide indexes 7, 8, 10, 11, 12, 13, 14, 15, 16, 17 are all width=0
→ Delete empty slides OR assign an image to each one
→ Also add this CSS to SFNT <style> block to prevent CLS while loading:

.slick-slider { min-height: 420px; visibility: hidden; }
.slick-initialized { visibility: visible; }
Issue 3 — srcset malformation: missing size descriptors
CriticalPerformanceConsole Error
Console: Failed parsing 'srcset' attribute value since it has an unknown descriptor. Dropped srcset candidate "graphics/00000001/8/GA". Images in sfnt_hero content section have srcset values with no size descriptor. The browser drops them entirely and falls back to src, loading the wrong size image.
📍 Content Management → Theme Components → sfnt_hero → each slide image srcset attribute

Fix — Add size descriptor to every srcset value

<!-- BEFORE (broken) -->
srcset="graphics/00000001/8/GA"

<!-- AFTER (fixed) -->
srcset="graphics/00000001/8/GA 800w,
graphics/00000001/8/GA-small 400w"
sizes="(max-width: 768px) 100vw, 1200px"
Issue 4 — No LCP image preload (LCP = 7.2s on mobile)
HighLCPPerformance
The hero banner image is the Largest Contentful Paint element. It is not preloaded, meaning the browser discovers it late during rendering. Lighthouse mobile LCP = 7.2s (target ≤2.5s). Adding a preload hint to the <head> is the single biggest LCP win available without a server change.
📍 SFNT template → <head> section → after line 32 (google-site-verification meta tag)

Fix — Add after <meta name="google-site-verification">

<!-- Step 1: Right-click hero image on live site → Open image in new tab → copy filename -->
<!-- Step 2: Replace YOUR-HERO-IMAGE.jpg with the real filename -->

<link rel="preload" as="image"
href="&mvt:global:basehref;graphics/00000001/YOUR-HERO-IMAGE.jpg"
fetchpriority="high">
Issue 5 — <style> block inside <section> (invalid HTML)
HighCode Quality
The .t-featured-categories__heading style block is placed inside a <section> tag in the body (lines 70–100). Style tags belong in <head> only. While browsers handle this, it can cause FOUC (flash of unstyled content) and fails HTML validation. It also runs on every page render unnecessarily.
📍 SFNT template → lines ~70–100 → <style> block inside <section class="o-layout">

Fix — Move the style block into the <head> <style> section

<!-- REMOVE from body (lines ~70-100): -->
<section class="o-layout">
<style> .t-featured-categories__heading { ... } </style>
</section>

<!-- ADD to <head> <style> block instead: -->
.t-featured-categories__heading { color:#1833B8; font-weight:700; ... }
Issue 6 — SearchSpring JS not deferred
MediumPerformance
The <div id="searchspring-finder-home"></div> div triggers SearchSpring JS to initialise on page load. SearchSpring's script is loaded globally via hdft without defer, meaning it blocks rendering. Adding defer to the SearchSpring script tag in hdft will allow the page to render first.
📍 hdft template → SearchSpring <script> tag → add defer attribute
<script src="https://snapui.searchspring.io/sinisterdiesel.js"></script>
<script src="https://snapui.searchspring.io/sinisterdiesel.js" defer></script>
Issue 7 — Featured products, promos, instagram all commented out
MediumConversion
Four content sections are wrapped in <mvt:comment> tags: sfnt-shop-by-brand, sfnt_promos, featured_products, instagram_feed. These are disabled features — no performance impact. However, featured_products being disabled means no product listings show on the homepage, which is a missed conversion opportunity.
📍 SFNT template → lines 97–107
<!-- These are disabled — no action needed for performance -->
<!-- To re-enable featured products, remove mvt:comment wrapper: -->
<mvt:comment><mvt:item name="readytheme" param="productlisting( 'featured_products' )" /></mvt:comment>
<mvt:item name="readytheme" param="productlisting( 'featured_products' )" />

Template: CTGY_BRANDS — Brand Landing Page

Brand category pages · 107 lines · Shows subcategories as product-style grid

Issue 11 — Category images missing width/height causing CLS
HighCLSPerformance
In CTGY_BRANDS, subcategory images use style="aspect-ratio: &mvte:...width; / &mvte:...height;" which is dynamic and relies on Miva loading the image dimensions. If Miva returns empty values, the browser has no reserved space and causes layout shift. The width="" and height="" HTML attributes should be set explicitly as a fallback.
📍 CTGY_BRANDS template → line ~75 → <img> tag inside subcategory loop

Fix — Add explicit fallback dimensions

<img alt="&mvte:subcategory:name;" class="x-product-list__image"
decoding="async" height="&mvte:top_category_image:image:height;"
loading="lazy" src="&mvte:subcategory:image;"
style="aspect-ratio: ..." width="&mvte:top_category_image:image:width;">

<img alt="&mvte:subcategory:name;" class="x-product-list__image"
decoding="async"
height="&mvte:top_category_image:image:height;"
width="&mvte:top_category_image:image:width;"
loading="lazy"
src="&mvte:subcategory:image;"
onerror="this.style.minHeight='150px'">

MediumReview

BSKE — Basket Empty & Abandoned Basket Email

🛒

Template: BSKE — Checkout: Basket Empty

Shown when cart is empty · 52 lines · Clean template, minimal issues

Issue 13 — Inline script runs synchronously on empty cart page
HighPerformance
The continueShopping() function is defined in an inline <script> tag inside the page body (line ~29). While small, inline scripts block HTML parsing. It should use defer or be moved to a separate JS resource. Also, the function reads sessionStorage which is safe but the implementation could be simplified.
📍 BSKE template → lines 28–34 → inline <script> in body

Fix — Add defer or move to resource file

<script>
function continueShopping(link) {
if (sessionStorage.getItem('continue_url')) {
link.href = sessionStorage.getItem('continue_url');
}
}
</script>

<!-- Move to Resources → JavaScript → create continue-shopping.js -->
<!-- Then load it with: -->
<script src="&mvt:global:basehref;js/continue-shopping.js" defer></script>
Issue 14 — No "Continue Shopping" button text guidance
MediumUXConversion
The empty cart page shows "Your shopping cart is currently empty" with a "Start Shopping" button. This is functional but a conversion opportunity is missed — there's no featured product suggestion, no category links, and no social proof. Adding 3–4 category quick-links or a "Popular right now" section here would recover some abandoned sessions.
