Exact Code Changes for Miva SFNT Template

FIX 1 — Hero Image Preload (add to <head> in SFNT, line ~33)

Paste this directly after the <meta name="google-site-verification"> line. Replace the href with your actual hero image URL (right-click hero on live site → Copy image address).

<link rel="preload" as="image"
     href="https://www.sinisterdiesel.com/mm5/graphics/00000001/YOUR-HERO-IMAGE.jpg"
     fetchpriority="high">

FIX 2 — Slick CLS Fix (add to <style> block in SFNT, after line 30)

Add these 3 lines immediately after the existing .slick-dotted.slick-slider block. This reserves carousel height before JS loads so the page doesn't jump.

/* CLS fix — reserves carousel space before Slick JS fires */
.slick-slider { min-height: 420px; }
.slick-slide { display: none; }
.slick-slide.slick-active { display: block; }

FIX 3 — Disable Affirm on Homepage (no code change needed)

In the SFNT template page in Miva, scroll to the bottom section "Affirm As Low As Messaging". Change Page Type from Homepage to None or uncheck it. This stops Affirm JS (145KB) loading on every homepage visit.

Page Type: Homepage  ← CHANGE THIS TO: None
Page Type: None      ← saves 145KB + removes SecurityError

FIX 4 — Fix Empty Slide 11 (in Content Management, not SFNT template)

Go to: Content Management → Theme Components → sfnt_hero
Find the slide that has no image assigned (slide #11 — it renders with slideWidth=0 causing CLS). Delete that empty slide item. This fixes the Slick carousel layout shift bug.

Location: Content Management → Theme Components → sfnt_hero
Action: Find empty slide (no image) → Delete it
Effect: Fixes slideWidth=0 CLS bug on carousel load

FIX 5 — Defer SearchSpring JS (in hdft template, not SFNT)

In Advanced → Templates → hdft, find the SearchSpring script tag and add defer:

<script src="https://snapui.searchspring.io/...js"></script>
<script src="https://snapui.searchspring.io/...js" defer></script>

⚠ Note: SearchSpring said it cannot be fully moved to GTM — but adding defer to the script tag is safe and does not break functionality. It just loads after the page renders instead of before.

FIX 6 — Category Image Alt Tags (in top_categories navigation set)

Go to: Content Management → Navigation Sets → top_categories
For each category image, add the alt text and explicit dimensions:

<img src="filtration.jpg">
<img src="filtration.webp"
     alt="Sinister Diesel Filtration Kits — Fuel Filter Conversion"
     width="300" height="200"
     loading="lazy">
