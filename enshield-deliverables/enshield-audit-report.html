Enshield — Website Audit03 · SEO & Accessibility
Search Readiness

SEO audit

Analyzed via Chrome DevTools. Fundamentals are solid — metadata and social cards are well configured — with a few structural fixes worth making.

Strengths

  • Title & description — clear and descriptive meta tags present.
  • Open Graph & Twitter cards — correctly implemented for social sharing.
  • Canonical URL — present, pointing to the main domain.
  • Viewport & lang — mobile viewport and en-US set correctly.

Fix these

  • Heading hierarchy — jumps H1 → H2 → H4, skipping H3. Break logical order for crawlers.
  • Empty image alt — the logo has alt=""; set alt="Enshield Logo".
  • Single H1 — good, but feature blocks should be H3 under their H2.
Inclusive Design

Accessibility audit

Working well

  • <nav> landmark present for primary navigation.
  • Icon links labelled — hamburger & cart use aria-label.
  • Language declared on the html element.

Needs attention

  • Missing <main> & <footer> landmarks — hurts screen-reader navigation.
  • Unlabelled links — several <a> tags have no text or aria-label.
  • Contrast — verify text hits WCAG AA 4.5:1, esp. on light theme (#e5f1f2).

Enshield — Website Audit04 · Action Plan
Prioritized

Recommendations & next steps

Ordered by impact-to-effort. Tackling the top three addresses the majority of the load-time and search issues.

Cut JavaScript weight

Audit the 51 scripts (1.34MB). Remove or consolidate unused third-party GoDaddy scripts — the biggest win for TTI (5.2s) and full load (16.4s).

HIGH
Set an efficient cache policy

Add long-lived cache headers to static assets — GTmetrix flags 368KB of savings available immediately.

HIGH
Add semantic landmarks

Wrap content in <main> and the base in <footer> for accessibility and SEO structure.

MEDIUM
Fix heading hierarchy

Change feature H4s to H3 so the outline flows H1 → H2 → H3 without gaps.

MEDIUM
Lazy-load below-fold images

Add loading="lazy" — currently 0 images use it — to speed initial paint.

MEDIUM
Add descriptive alt text

Set the logo alt to "Enshield Logo" and label remaining icon-only links.

LOW
Resolve layout shifts & add a CDN

Fix the 2 CLS-causing shifts (reserve space for media) and serve static assets via CDN.

LOW
The strategic takeaway: enshield.com already scores an A on raw metrics — but nearly every remaining issue stems from the GoDaddy builder's script bloat and rigid structure. Migrating to the lightweight, purpose-built front end (the React mockup) resolves the caching, JS-weight, semantic-HTML, and heading issues in one move — while unlocking the premium look and animation the brand deserves.
