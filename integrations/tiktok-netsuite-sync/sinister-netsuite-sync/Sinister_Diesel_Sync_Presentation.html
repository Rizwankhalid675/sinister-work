Overview

What Is Sinister Diesel Sync?

Sinister Diesel Sync is a fully custom-built Windows background service that automatically bridges our Miva storefront (sinisterdiesel.com) and our NetSuite ERP — replacing the $17,797/year Celigo subscription with an in-house solution we own completely.

It runs silently in the background on our Windows server, wakes up every 5 minutes, and executes 5 automated flows covering everything from order creation to invoice closing.

Smart SKU Matching
Checks hardcoded overrides → exact SKU match → strips suffixes for partial match → auto-creates the item in NS if still not found. No order ever loses a line.
💰
Discount / Promo Codes
Discount amount from Miva is mapped to the NS Discount Item and Discount Rate header fields. Sales reps can see the promo value at a glance on every order.
🔒
Duplicate Prevention
Every synced order ID is saved to synced_orders.json. On each sync cycle, the system checks this file first — an order is never created twice in NetSuite, even if it appears in multiple sync windows.

Flow 03

Deposits & Invoices: The Financial Flow

This is the most critical flow — it handles all the financial transactions in NetSuite for every order, from payment capture to invoice closure. Everything is fully automated.

1
Order Placed on Miva
Customer completes checkout. Full payment is collected at time of purchase.
2
Sales Order Created in NetSuite
Flow 1 runs — a Sales Order is created in NS with status Pending Approval (A).
3
Customer Deposit Created
A Customer Deposit is created immediately, linked to the Sales Order, for the full order amount. This records the payment received.
4
Order Approved & Fulfilled
The team approves the order and creates an Item Fulfillment in NS. SO status moves to E or F (Pending Billing).
5
Invoice Auto-Generated
The sync detects the Pending Billing status and automatically transforms the Sales Order into an Invoice in NetSuite.
6
Invoice Auto-Closed ✅
The Customer Deposit from Step 3 is applied to the Invoice — balance becomes $0.00 and the invoice is marked CLOSED. No manual payment entry needed.
⚠️ Invoice is NEVER created until the Sales Order is approved AND fulfilled — this matches real-world fulfillment timing exactly

System Architecture & File Structure

sinister-netsuite-sync/
  │
  ├── index.js             ← Main entry point · scheduler · runs all 5 flows
  ├── miva.js              ← Miva API client (orders, shipments, products)
  ├── netsuite.js         ← NetSuite REST API client (TBA OAuth 1.0a)
  ├── logger.js           ← Timestamped logging to console + sync.log
  │
  ├── flows/
  │   ├── ordersToNetsuite.js  ← Flow 1: Miva orders → NS Sales Orders
  │   ├── shipmentsToMiva.js   ← Flow 2: NS shipments → Miva tracking
  │   ├── invoices.js         ← Flow 3: Deposits + Invoice + Auto-Close
  │   ├── productSync.js      ← Flow 4: Product ID sync
  │   └── customersToNetsuite.js ← Flow 5: Miva customers → NetSuite
  │
  ├── logs/
  │   ├── sync.log            ← Full timestamped activity log
  │   ├── synced_orders.json   ← Tracks synced Miva → NS orders
  │   └── synced_invoices.json ← Tracks deposit + invoice per order
  │
  └── .env                ← API credentials (never share)

POST requests to sinisterdiesel.com/mm5/json.mvc with X-Miva-API-Authorization header. Store Code: SD.
